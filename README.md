# ActivoFit: Продуктовая аналитика мобильного приложения

**Учебный проект.** ActivoFit — вымышленное фитнес-приложение с подпиской.
Все данные синтетические, сгенерированы для отработки навыков продуктовой аналитики.

## Стек
- PostgreSQL
- SQL (оконные функции, JOIN, агрегации, подзапросы)
- DBeaver
- DataLens (дашборд — в процессе)

## Данные
База данных содержит три таблицы:
- `users` — 1000 пользователей: страна, платформа, источник трафика, дата установки
- `events` — ~6000 событий: установка, регистрация, онбординг, триал, покупка, сессии
- `subscriptions` — ~100 подписок

## Цель исследования
Оценить эффективность онбординга, сравнить конверсию по платформам и источникам трафика, посчитать Retention пользователей.

## Основные вопросы
- Как работает онбординг? Сколько пользователей доходят до покупки, где самая большая потеря?
- Отличается ли конверсия на iOS и Android? Возможно, на одной из платформ проблема.
- Какие источники трафика приносят больше платящих пользователей? Чтобы понять, куда вкладывать маркетинговый бюджет.
- Какой Retention? Возвращаются ли пользователи после установки?

## Ключевые выводы 
- Конверсия из установки в покупку - 10,3%. Самая большая потеря - в момент перехода из пробного периода к покупке: только 27% пользователей, активировавших пробный период, становятся платящими. Рекомендация: провести А/В тест механик повышения конверсии (напоминания об окончании пробного периода, скидка на первый месяц)
  
- Платформа не является ключевым фактором конверсии, разница между iOS и Android составляет менее 1 п.п. Проблема на стороне продукта, а не технической реализации под платформу
  
- Органический трафик, TikTok и Google Ads — самые эффективные каналы с точки зрения конверсии в покупку. Facebook Ads приводит больше всего установок, но конвертирует хуже (9%), что может указывать на низкое качество аудитории. Referral — самый слабый канал (7%). Рекомендация: перераспределить бюджет в пользу органического продвижения, Tiktok и Google Ads, пересмотреть креативы для Facebook Ads, разработать реферальную программу для повышения конверсии referral
  
- Retention сильно падает уже на 4-ый день после установки приложения: активность снижается до 25-40 пользователей в день. Обнаружены аномальные выбросы по активности на 10ый день - 125 человек, 26-й - 50, возможно в эти периоды проводились какие-то рассылки/акции. Рекомендация: проверить коммуникацию с пользователями в эти периоды, оценить их влияение на Retention и рассмотреть возможность введения стимулирующих механик на регулярной основе
