# ActivoFit: Продуктовая аналитика мобильного приложения

**Учебный проект.** ActivoFit — вымышленное фитнес-приложение с подпиской.
Все данные синтетические, сгенерированы для отработки навыков продуктовой аналитики.

## Стек
- PostgreSQL
- SQL (оконные функции, JOIN, агрегации, подзапросы)
- DBeaver
- DataLens (дашборд — в процессе)

## Данные
База данных содержит три таблицы:
- `users` — 1000 пользователей: страна, платформа, источник трафика, дата установки
- `events` — ~6000 событий: установка, регистрация, онбординг, триал, покупка, сессии
- `subscriptions` — ~100 подписок

## Цель исследования
Оценить эффективность онбординга, сравнить конверсию по платформам и источникам трафика, посчитать Retention пользователей.

## Основные вопросы
- Как работает онбординг? Сколько пользователей доходят до покупки, где самая большая потеря?
- Отличается ли конверсия на iOS и Android? Возможно, на одной из платформ проблема.
- Какие источники трафика приносят больше платящих пользователей? Чтобы понять, куда вкладывать маркетинговый бюджет.
- Какой Retention? Возвращаются ли пользователи после установки?
